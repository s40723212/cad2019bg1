Solidworks

開發緣起

1993年

創始人Jon Hirschtick招募了一個工程師團隊，旨在使3D CAD軟件更容易上手。Hirschtick從廣受好評的麻省理工學院二十一點隊的成員中籌得100萬美元，開始了他的新事業。該團隊的目標是構建一個基於Windows平台的易於使用且價格合理的軟件。這在當時真的是革命性的存在，當時所有其他流行的CAD系統都是在Unix上構建的。經過幾年的開發，最初發佈的SolidWorks終於來了......

1995年

  • 與其他CAD系統的18,000美元相比，SolidWorks的售價僅為4,000美元。
  • 該軟件更易於使用：與市場上其他CAD軟件相比，用戶只需3個月的使用經驗就可以輕鬆上手。易用性的很大一部分原因在於它採用了現代着名的Windows界面。
  • 模型的可視化是前所未有的。在光線充足的陰影環境中的構建功能與傳統建模空間形成鮮明對比，傳統建模空間僅顯示黑色背景下的線框。
  • 我們現在熟悉的類似FeatureManager用於顯示模型的構建歷史。
  • 零件，組件和圖紙從一開始就可用。用戶無法在組件中應用配合，但仍可將組件放置到位。

1996年

  • 介紹了上下建模。允許使用自上而下的裝配建模方法。
  • 用戶現在可以在裝配中使用配合併捕捉動態裝配運動。除了干擾檢測，用戶還可以更輕鬆地驗證形狀，適合度和功能。
  • 拖放功能可將一個裝配體零部件移動到另一個裝配體中。
  • 部分配置。
  • 自動化物料清單。只有一種方法沒有自定義選項，但它很有效。

1997年

  • 鈑金功能首次發佈。
  • 標準庫功能推出。
  • Loft功能現在可以使用引導曲線。
  • SolidWorks 被 Dassault Systemes（達索公司）收購。

1997+

  • 面部圓角介紹。
  • 裝配配置。
  • 完全可定製的組件爆炸視圖。
  • SolidWorks Viewer作為免費產品推出，可與非CAD用戶共享設計，並在與外部方共享數據時保護知識產權。

1998年

  • 已發佈功能面板，現在與設計庫的功能類似。可以拖放到模型上的預製功能。
  • 草圖診斷可幫助用戶理解草圖不適用於特定特徵的原因。
  • Lofts和Sweeps的相切控制。
  • 組合件封包(Assembly Envelopes)。

1998+

  • 智能配合在建造組件中更多地使用拖放式自動化。
  • 隨着用戶不斷建造越來越大的組件，輕量級組件可以幫助提高性能。
  • 表面處理工具首次推出。
  • FeatureWorks用於導入文件，能夠更有效地使用來自其他軟件的現有3D數據。
  • 首次CSWP考試發佈。它是手寫的，並通過郵件發送。

1999年

  • Palm Springs的第一個SolidWorks World。
  • 拖動組件時對組件的實時碰撞檢測。
  • 3D草圖介紹。
  • 管道模塊發佈。
  • 2D命令模擬器有助於縮小與以前AutoCAD用戶的差距。

2000年

  • 發佈eDrawings。
  • 實現了Hole Wizard接口。
  • 表面處理更有用，能夠編織，修剪和延伸表面。
  • SolidWorks Explorer。
  • 動態間隙檢測可驗證裝配部件之間的最小間隙。

2001年

  • 鏡像組件中的組件。
  • 引領互動。以前，在任何CAD系統中，只要選擇了命令，軟件就會在模型頂部顯示整個對話框。SOLIDWORKS開始廢除這一點，並轉向我們現在知道的PropertyManagers，並將陰影標註返回到模型。
  • 現在允許靈活的子組件進一步捕捉準確的裝配運動。
  • 介紹填充表面命令。
  • DXF / DWG導入嚮導發佈。
  • 3D Meeting（3D會議），最終在幾個版本之後退出。利用Microsoft Live Meeting並允許與螢幕共享進行實時同步會議。

2001+

  • 運動模擬功能現在可以捕捉組件中移動的組件的實際物理動態。
  • 介紹大型裝配模式。
  • 實現了2D到3D轉換工具，以便將2D CAD數據輕鬆轉換為功能齊全的3D模型。
  • 介紹SolidWorks Office，將常用的插件如PhotoWorks，Toolbox，Utilities和FeatureWorks打包成一個產品。
  • Dassault Systemes收購了SRAC並開始將仿真引入主流設計流程。

2003年

  • COSMOSXpress是第一款Xpress產品，它為每個桌面帶來了簡單的零件驗證。
  • 3DContentCentral為用戶提供了一個交互和共享模型的社區。
  • 多體零件設計。這是一個非常大的問題，現在是焊接，鈑金和模具設計的基礎。
  • 收購了PDMWorks並引入了SolidWorks Office Professional軟件包。開始將PDM作為每個客戶提供的一部分。

2004年

  • 界面更新，包括CommandManager，彈出FeatureManager和PropertyManager和ConfigurationManager的管理器視圖選項卡。
  • 介紹了Mold Tools（模具工具）和Weldments（焊件）。
  • 實施Auto Balloon命令以匹配物料清單物料編號中的balloon編號。
  • 限制配合以捕捉更複雜的裝配運動。
  • 推出了RealView Graphics，以便隨時提供更逼真的圖像。

2005年

  • 繪圖比較，為用戶提供工具來比較兩個圖紙之間的變化。
  • Flex功能。
  • 電氣布線，與管道布線一起使用。
  • 在圖紙中自動標註，以捕捕捉模型中已定義的尺寸。
  • 能夠繪製無限的線條。
  • SolidWorks Rx發佈，旨在提供幫助技術支持更好地去識別問題。

2006年

  • 發佈Smart Components功能，允許為硬件之類的物件安裝智能裝配組件。
  • 介紹填充圖案。
  • 介紹草圖塊。
  • 顯示狀態。
  • 設計檢查器，允許用戶根據公司標準動態檢查他們的模型。
  • 添加了相機視圖功能。
  • 發佈SolidWorks Office Premium（白金版）軟件包，包括Routing，COSMOSWorks和 COSMOSMotion進行模擬，以及ScanTo3D。

2007年

  • SWIFT工具可幫助設計人員在提供自動化解決方案時動態了解模型存在問題或錯誤的原因。
  • 可在組件中使用皮帶，鏈條和齒輪功能。
  • 用於表面處理的自由形式命令。
  • 介紹Enterprise PDM。

2008年

  • 使用新的菜單欄和CommandManager佈局對用戶界面進行全面檢修和現代化優化。上下工具欄和快捷工具欄首次可用。所有這些界面組件現在都可以完全自定義。
  • Instant3D，提供拖動和操縱幾何體的功能，可輕鬆測試不同的設計概念。
  • 發佈TolAnalyst。
  • 設計剪貼畫，通過剖析現有數據以便在另一個模型中重複使用，可以更輕鬆地重用現有的2D和3D數據。
  • 推出DriveWorksXpress和DFMXpress。

2009年

  • PhotoView360圖像渲染作為第一個渲染器引入，以訪問處理器的多核功能。
  • SpeedPak現在可用於簡化裝配中的組件。
  • 為追求更高目標，所以引入傳感器。
  • CircuitWorks發佈並添加到SolidWorks Office Premium（白金版）軟件包中。
  • 發佈3DVIA Composer。

2010年

  • 現在提供可持續發展方案，幫助用戶了解其設計對環境的影響。
  • 快速標註尺寸使得創建細節圖更加容易。
  • 介紹基於事件的運動模擬。
  • 首次提供鼠標手勢快捷鍵。
  • 現在可以進行多體鈑金建模。

2011年

  • PhotoView360完全集成到SolidWorks界面中。
  • 現在可以使用Walk-through（走查）功能。
  • 引入了Defeature工具以幫助限制模型中的知識產權數量。
  • 現在可以在Simulation中使用平面簡化。
  • 自動排列圖紙中的尺寸。

2012年

  • 方程式編輯器經過徹底檢修後，更加用戶友好。
  • 介紹Large Design Review。
  • Feature Freeze現在可以讓用戶更好地控制性能。
  • 鈑金增強：邊線法蘭的更多選擇，引入掃描法蘭，使用的成型工具更加人性化。
  • 發佈成本計算加載項。

2013年

  • View Orientation有一個對話框，不再只是在工具欄上了。還引入了選擇器多維數據集以及保存視圖功能以便之後使用。
  • 以前的版本文件互操作性：2013文件現在可以在2012 SP5中打開。
  • 介紹Conic草圖實體。
  • 相交功能。
  • Customer Portal（客戶門戶網站）上提供了CAD Admin Dashboard（CAD管理員儀錶板）。

2014年

  • 能夠通過上下工具欄快捷方式添加標準配合。
  • 替換草圖實體使用戶能夠用新的實體替換舊實體並仍然維護下游引用。
  • 推出了鈑金中的Lofted Bend功能。
  • 能夠創建Conic Fillets（圓錐倒角）。
  • 樣式樣條草圖命令。
  • 自動縮放第一個草圖。現在，在草圖上創建第一個維度時，所有實體都會自行調整大小以進行縮放。
  • 能夠配置結構成員配置文件。現在允許配置輪廓草圖，這大大改變了需要組織的文件數量。
  • MySolidWorks作為能夠提供自學培訓以及所有SolidWorks的在線資源社區發佈。

2015年

  • 動態參考可視化，幫助用戶了解父/子關聯。
  • 引入了選擇集，以便於重複使用幾何選擇組合。
  • 能夠直接從SolidWorks進行3D打印。
  • Simulation中的疲勞分析。
  • Profile center mate（輪廓中心配合）。
  • 發佈SolidWorks MBD。

2016年

  • 用戶界面重新設計，包括更好地支持高解像度顯示器。
  • 藍色/灰色盤默認採用傳統配色方案。
  • 引入了選擇麵包屑，使相關命令更易於訪問。
  • 實現Mate Controller工具，以更加可訪問的方式捕捉裝配定位。
  • 能夠全局替換失敗的配合實體。
  • 介紹Thread功能。
  • 發佈PDM Standard（標準版）並添加到SolidWorks Professional專業版軟件包中。
  • SolidWorks Visualize作為獨立的圖片渲染應用程式發佈。

2017年

  • 引入3D互連以簡化不同文件類型與SolidWorks的協作。
  • Asset Publisher具有ground plane（地層）和magnetic mates（磁配對），可輕鬆實現大型裝配佈局。
  • 引入了Advanced Hole（高級孔）功能。
  • 增強了Wrap功能，可在更多類型的面選擇上創建幾何體。
  • 陰影草圖輪廓增強選擇和操作功能。

2018年

  • SolidWorks啟動時的歡迎對話框。
  • 用戶設置現在可以登錄。新發佈的在線許可也可以遵循此登錄，在需要在多台計算機上使用SolidWorks時為用戶提供另一種選擇。
  • 選擇幾何圖形，允許用戶忽略幾何圖形並在其上選擇其他實體。
  • 引入了標籤和插槽功能，自動執行以前在鈑金和焊件中常用的手動方法。
  • 能夠鏡像3D草圖中的實體。
  • 首次使用筆，觸屏和基於手勢的草圖繪製功能。
  • 引入拓撲仿真分析。

2019年

  • 顯著改進裝配性能。
  • Silhouette defeature命令，簡化了從模型中刪除細節以提高性能或保護知識產權的過程。
  • 能夠按狀態對mates進行分組。
  • 部分倒角和圓角。
  • 將圖像轉換為網格/凹凸貼圖，從圖像文件創建實際的3D幾何體。

NX

開發起源

1960年

McDonnell Douglas Automation 公司成立。

1976年

收購了Unigraphics CAD/CAE/CAM系統的開發商——United Computing公司，UG的雛形問世。

1983年

UG 上市。

1986年

Unigraphics吸取了業界領先的、為實踐所證實的實體建模核心——Parasolid的部份功能。

1989年

Unigraphics宣佈支持UNIX平台及開放系統的結構，並將一個新的與STEP標準兼容的三維實體建模核心Parasolid引入UG。

1990年

Unigraphics作為McDonnell Douglas（波音飛機公司）的機械CAD/CAE/CAM的標準。

1991年

Unigraphics開始了從CAD/CAE/CAM大型機版本到工作站版本的轉移。

1993年

Unigraphics引入複合建模的概念，可以實體建模、曲線建模、框線建模、半參數化及參數化建模融為一體。

1995年

Unigraphics首次發佈了Windows NT版本。

1996年

Unigraphics發佈了能自動進行干涉檢查的高級裝配功能模塊、最先進的CAM模塊以及具有A類曲線造型能力的工業造型模塊：它在全球迅猛發展，佔領了巨大的市場份額，已經成為高端及商業CAD/CAE/CAM應用開發的常用軟件。

1997年

Unigraphics新增了包括WAVE（幾何連結器）在內的一系列工業領先的新增功能。WAVE這一功能可以定義、控制、評估產品模板，被認為是在未來幾年中業界最有影響的新技術。

2000年

Unigraphics發佈了新版本的UG17，最新版本的，是UGS成為工業界第一個可以裝載包含深層嵌入「基於工程知識」（KBE）語言的世界級MCAD軟件產品的供應商。

2001年

Unigraphics發佈了新版本UG18，新版本對舊版本的對話框進行了調整，使得在最少的對話框中能完成更多的工作，從而簡化了設計。

2002年

Unigraphics發佈了UG NX1.0.新版本繼承了UG18的優點，改進和增加了許多功能，使其功能更強大，更完美。

2003年

Unigraphics發佈了新版本UG NX2.0 。新版本基於最新的行業標準，它是一個全新支持PLM的體系結構。EDS公司同其主要客戶一起，設計了這樣一個先進的體系結構，用於支持完整的產品工程。

2004年

Unigraphics發佈了新版本的UG NX3.0，它為用戶的產品設計與加工過程提供了數字化造型和驗證手段，。它針對用戶的虛擬產品的設計和工藝設計的需要，提供經過實踐驗證的解決方案。

2005年

Unigraphics發佈了新版本的UG NX4.0.它是嶄新的NX體系結構，使得開發與應用更加簡單和快捷。

2007年04月

UGS公司發佈了NX 5.0 – NX的下一代數字產品開發軟件，幫助用戶以更快的速度開發創新產品，實現更高的成本效益。

2008年06月

Siemens PLM Software發佈UG NX 6.0，建立在新的同步建模技術基礎之上的NX 6將在市場上產生重大影響。同步建模技術的發佈標誌着NX的一個重要里程碑，並且向MCAD市場展示Siemens的鄭重承諾。 NX 6將為我們的重要客戶提供極大的生產力提高。

2009年10月 

西門子工業自動化業務部旗下機構、全球領先的產品生命周期管理（PLM）軟件與服務提供商Siemens PLM Software 宣佈推出其旗艦數字化產品開發解決方案NX 軟件的最新版。UG NX 7.0引入了「HD3D」（三維精確描述）功能，即一個開放、直觀的可視化環境，有助於全球產品開發團隊充分發掘PLM信息的價值，並顯著提升其制定卓有成效的產品決策的能力。此外，NX 7.0還新增了同步建模技術的增強功能。修復了很多6.0所存在的漏洞，穩定性方面較6.0有很大的提升。

2010年5月20日

Siemens PLM Software在上海世博會發佈了功能增強的NX7最新版本（UG NX 7.5），NX GC 工具箱將作為NX 7最新版本的一個應用模塊與NX 7一起同步發佈。NX GC 工具箱是為滿足中國用戶對NX特殊需求推出的本地化軟件工具包。在符合國家標準（GB）基礎上，NX GC 工具箱做了進一步完善和大量的增強工作。

2011年09月 

Siemens PLM Software發佈了UG NX 8.0

2012年10月 

Siemens PLM Software發佈了UG NX 8.5

2013年10月

Siemens PLM Software發佈了UG NX 9.0

2014年8月

Siemens PLM Software發佈了UG NX 10.0

2016年8月

Siemens PLM Software發佈了UG NX 11.0

2017年10月

Siemens PLM Software發佈了UG NX 12.0

開發起源

Autodesk Inventor軟件是美國AutoDesk公司於1999年底推出的三維可視化實體模擬軟件，目前已推出最新版本Inventor 2017，實驗室使用的是Inventor 6。它包含三維建模、信息管理、協同工作和技術支持等各種特徵。使用Autodesk Inventor可以創建三維模型和二維製造工程圖、可以創建自適應的特徵、零件和子部件，還可以管理上千個零件和大型部件，它的“連接到網絡”工具可以使工作組人員協同工作，方便數據共享和同事之間設計理念的溝通。Inventor在用戶界面簡單，三維運算速度和著色功能方面有突破的進展。它是建立在ACIS 三維實體模擬核心之上，設計人員能夠簡單迅速地獲得零件和裝配體的真實感，這樣就縮短了用戶設計意圖的產生與系統反應時間的距離，從而最小限度的影響設計人員的創意和發揮。

特點:
1.正確的設計工具:
     Inventor產品線提供了一組全面的設計工具，支持三維設計和各種文檔、管路設計和驗證設計。Inventor不僅包含數據管理軟件、AutoCAD Mechanical的二維工程圖和局部詳圖，還在此基礎上加入與真正的DWG兼容的三維設計。Inventor提供的專家系統能以最快捷的方式生成製造用工程圖，從而加速了從草圖到成品的過程。
2.滿足設計需要的專用工具:
     專用工具能夠創建和驗證管路系統設計，包括管材、管件和線束設計，從而節省時間並降低創建原型的成本。Autodesk Inventor Professional提供的工具能夠創建完整的，包括複雜的管路系統的設計，同時自動創建精確的物料清單和完整的製造文檔。
在創建之前進行設計驗證。使用Autodesk Inventor Professional，可以模仿一個完整設計中各個組成部分的運動狀態，並且準確地預測瞬間載荷和加速度。此外，集成的有限元分析工具能夠幫助工程師完成應力應變和模態分析，以避免相關設計錯誤。
Inventor根據不同的軟件產品配置，可提供不同的功能，從而使其成為AutoCAD用戶在製造過程中的最佳選擇。
3.數字原型:
     創建數字原型以確定設計細節，節省製造真實零部件的成本。
4.（增強功能）三維虛擬模型:
      在設計初期驗證和模擬設計結果，能創建創新的、高質量的產品。同時降低製造成本和縮短進入市場的時間。
在完成實際機器之前，就完成對零件和裝配的構思和功能設計驗證。
在設計過程中或完成設計後，能夠簡潔地察看草圖、零件和裝配。同時在設計過程中，幫助設計者選擇最佳的設計方案。
使用裝配位置表達，在不同機構運動位置評估設計結果。 [
5.干涉與接觸檢測:
     使用Inventor裝配檢測工具減少設計錯誤並提高設計的工藝性。
使用自動化工具測試零件在裝配中的干涉。
拖動某一個零件使其與另一個零件接觸以驗證兩者之間的關係。
在一組接觸部件中選擇獨立的組成以確定該組成是否進行預想的規則運動。
6.自動限制:
    自動極限功能將自動監測選定的設計參數並在參數超出指定的範圍時顯示顏色標識警告以減少設計錯誤。使用自動限制功能可以監測長度、距離、角度、直徑、周長、面積和質量。
在Inventor裝配環境下可將裝配模型存為STL格式的文件以應用於快速成型。
7.物理屬性:
    利用真實屬性創建的數字原型可以完善產品的設計。在Inventor中創建的零件和裝配都帶有物理屬性信息包括重心、材料、密度、顏色和紋理。這些物理屬性信息能夠幫助設計者完整表達設計意圖。
8.運動仿真:
    使用Inventor的設計者，可以利用運動仿真預測產品在真實條件下的運動狀況。運動仿真可以減少創建真實零部件進行台架試驗的時間和成本，也可以避免昂貴的諮詢費用。
9.（增強功能）模擬:
     模擬機械裝置和原動力的運轉，以保證設計的正確性，同時也降低了創建真實零部件的費用。計算在整個運轉週期內的約束條件，並且精確地調整原動力和結構的參數，加載符合實際的驅動載荷。同時可以分析機械裝置中每一個零部件的位置、速度、加速度和載荷。 
10.（增強功能）輸出到FEA:
    將連續時間段中選定時間點的工況傳送到Autodesk用精確的工況條件進行應力應變分析並得出結果；據此調整零部件的尺寸形狀，將材料確實用到真正需要的地方，而成本降到最低。
11.（增強功能）定義負荷:
     用載荷編輯器，加載不同的驅動載荷和力矩以及基於時間的力函數，確認設計結果在不同的載荷條件下的性能。
12.（增強功能）可視性:
     三維動畫型的結果表達，將根據基礎實體和工況、約束進行運動仿真。因此能充分的觀察到驗證設計的運動和性能。
13.（增強功能）點軌跡:
     根據精確的部件位置確定某結構和固定結構間的間距。選擇模型中任意一點使用“軌跡”選項能記錄在動作全程中每一瞬間該點的位置。保存零件和裝配設計模擬的輸出結果中將包括點軌跡和裝配位置。
14.運動仿真曲線圖:
     使用所提供的相關圖表可以快速地表現出動態特性在機器的運轉週期中的變化。這些特性包括位置、力和時間加速度。使用同一圖表的多個副本可以比較模擬週期內每一點的不同特性。 
15.Microsoft Excel輸出:
     將運動軌蹟的XY 數據輸出到Microsoft Exce表格中進一步分析模擬結果並且把分析結果合併到產品設計結果的演示和報告中。
16.（增強功能）調用約束:
     可以簡捷創建運動仿真並描述機構的運動過程。運動約束會分析出模擬過程中的相關零件和正確的運動副。也能先添加連接約束庫中的標準運動約束。
然後添加彈簧和阻尼器。能定義每一個連接處的摩擦係數。
17.應力分析:
     使用Inventor可通過應力分析確認零件在載荷下的變形等狀況，以確認設計的合理性。
18.評估零件性能:
     使用評估功能檢測零件在真實操作條件下的變形使用材料選擇分析，為零件選擇最合理的材料種類和位移狀況，省略了傳統的計算和試驗，節省了零件進行決策支持。
19.材料選擇分析:
     選用的材料越少，每個零件的成本設計時間。就越低。同時還能節省運費、材料加工費和倉儲費。
20.安全係數:
     在分析的基礎上作出設計決策，並且協同運動仿真創建最合適的零件。能直觀的表達安全係數超值的並為這些區域制定備用方案。
21.（增強功能）應力分析:
     在Inventor中，借助ANSYS DesignSpace可直接進行應力和疲勞分析。內嵌機制可分析的類型包括：變形分析、安全係數分析、應力分析和模態分析。其中應力分析包括最大和最小主應力分析。
22.（增強功能）協同運動仿真:
     分析零件在其運動過程中不同時間點的工況，並且導入這個運動仿真下的結果參數，然後進行分析， 觀察在這個工況下的結果。
23.（新增功能）模型簡化:
     在有限元分析中，能根據需要抑制對於分析結果可以忽略的結構特徵，縮短分析計算的時間。
24.無摩擦約束:
     無摩擦約束使得零件分析的約束更為真實。例如銷子能夠在孔內旋轉， 零件可沿面進行移動。
25.（新增功能）薄壁件分析:
     能識別鈑金等薄壁零件中的高應力區域，並且確保薄壁金屬零件可以承受運動載荷。
26.易於使用的綜合分析:
      可以檢查零件設計和研究零件更改的影響，但不需要中斷零件設計過程或者退出Inventor 。分析過程並不需要對模型數據進行單獨的轉換。所以綜合分析功能比獨立方案更易於使用。能直接訪問一般是為分析師設置的功能，因而不再依賴專家進行分析的過程。  
27.在裝配環境下分析零件:
      能在裝配環境下進行零件分析，而不必打開這個零件。這樣的分析模式會使設計者更好的了解零件在機構與裝配中的情況。
28.（增強功能）深入的結果觀察:
      因為分析計算工況和結果是依附於模型的，就可以修改模型並回到分析環境，再次進行運動仿真，獲得新的計算結果。
29.將分析數據輸出到ANSYS:
      Inventor的分析可以直接應用於其他ANSYS產品中，並且為工程檢測和進一步的研究提供基礎。這個功能還可以準確地把數據傳遞到更高級的ANSYS模擬產品中。 [3] 
30.變形動畫:
      對變形的研究會更深入地了解零件在外力作用下的反應。這個結果可以製作成動畫，並且存儲為AVI格式的文件。
31.共享驗證結果:
      輸出AVI或者圖形格式文件，能順利地將分析結果添加到報告中。
32.管路設計:
      使用Inventor的設計管理可以節約創建管路和軟管的時間。
33.管路:
      可以選擇一個起點、一個終點和任意數量的中間點定義管路，便捷地創建和修改管路。它與工程圖的完全關聯使得三維裝配發生更改時，管路也會隨之自動更新。管路的位置可以用三維草圖工具，對受約束的管路精確控制。這種方法可以使管路與設計規則在創建和修改管路的過程中保持一致，例如最小和最大長度標準。
34.自定義管路彎曲:
      在實際的管路設計中，可能需要彎曲一個管路，而不是插入一個彎頭。自定義管路彎曲功能具有靈活性和控制力，即使對非標准設計也有效，可以按照創建剛性管路的方法創建具有自定義彎曲半徑和角度的管路。
35.軟管:
     創建與三維數字原型關聯的精確的製造文檔，確保軟管和配件的完全匹配。系統會從資源中心調出合適的軟管，並且根據所選的軟管類型檢查最小折彎半徑。軟管的長度會自動更新，並在長度累加命令中使用。
36.（新增功能）法蘭管:
      當管路是法蘭連接組成時，確保使用正確的配件和墊片。Inventor會自動使用法蘭配件組裝管路，並且將接頭處替換為法蘭配件和對應的墊片。
37.剛性管路:
      可對形狀、彎曲角度和半徑的有效控制，快捷地創建剛性管路。剛性管路還可以根據任意數量的彎曲以及多種自定義的彎曲角度和半徑創建。其中對於半徑和角度參數會加強對剛性管路形狀的控制。
38.管件庫:
      能自動從豐富的管件庫中選取正確的管件進行放置，這就提高了設計質量，能順暢地組織零件而避免繁雜的搜索過程。這個配件庫包括了常用的行業標準，如：ANSI、DIN、ISO和JIS配件、管道和軟管。也可以在庫中添加或者編輯管件的屬性，包括現有零件編號、以及管理用於配件、管道和其他標準件的文件名。 [3] 
39.（新增功能）焊接管:
      定義焊接管時，需要創建精確的管路長度。Invntor 能自動使用焊接方式件組裝管路，並且會自動調整每一段管路的長度，留出正確的焊縫間距。
40.創建管路:
     自動使用與製造標準相符的真實零件組裝管路。這個工具將管路和軟管轉化成實體管路，在這條管路上自動放置所需的配件、管段、剛性管和軟管。在此過程中，將創建Inventor 的標準零件，以便快捷完成質量計算和乾涉檢測。此外，當管路長度為最大長度時，將自動放置中間接頭，用戶也可以指定自己的截斷長度。
41.ISOGEN 管路文件輸出:
     這是為在第三方應用軟件中自動創建管路設計的軸測工程圖。按行業規則表達管路設計結果。ISOENG管路文件是一個行業標準軟件Alias ISOGEN格式的文件。根據這種文件，ISOGEN軟件將創建一個包含軸測工程圖的DXF?或者DWG文件。 [3] 
42.裝配文檔:
      能快捷地將準確詳細的管路和軟管位置信息包含在裝配文檔中。由於所有的管道幾何圖元都是Inventor 自帶的，所以可以使用標準的工程圖功能創建裝配工程圖。
43.（增強功能）管路樣式:
      增強了管路質量和製造支持，確保自動創建的管路始終與符合設計標準。創建管路樣式支持螺紋，焊接和法蘭接頭的使用。根據設計規則為自動或者手動管道配置相應的配件。設計規則包括最小管段長度和最小半徑，以及兩個接頭間的最大長度。 [3] 
44.管路折彎表:
      避免錯誤。創建XYZ 或者YBC 格式的無關聯的ASCII管路折彎表，這個表為製造提供了可靠的資料。
45.支管配件:
      擴展管道模型包括螺紋支管配件和焊接支管配件。使用交互式控件從資源中心調用支管配件並將其安放到現有管路上，可定義位置和角度。使用這種方法添加的支管配件會自動在現有管子上生成正確的附加孔。
46.複製管路:
     通過便捷的複制，重複使用以前設計中的完整管路， 提高工作效率。
47.佈線設計:
      使用Inventor 進行佈線設計能夠節省時間和資源。佈線設計中包括了帶狀電纜的設計。
48.智能導線創建:
      保持電路圖與三維導線線束設計的一致性，以便簡化導線設計，並且減少製造失誤。在Inventor中，導線清單和接頭通過內嵌的電子和實體模型數據的交互檢測來控制線束設計的。因此所有的導線和接頭都會顯示在電纜設計的三維模型中。
49.導入導線清單:
     在裝配環境下導入導線清單的同時保留電路圖設計，可以減少和降低設計錯誤。可以從Auto C AD Electrical 或者第三方方案設計應用軟件中導入大量的導線清單，並且檢測和添加遺失的接頭、接插件的針和導線定義。
50.線束路徑定義:
      優化了電纜和線束部件的設計，確保留出製造需要的空間以減少由於不完整的設計定義引起的製造錯誤。使用選擇一點後單擊的方法來定義三維虛擬套管，並且可以通過添加點或移動現有點重新定義線束的形狀。然後創建關聯以確保設計部件更改時自動更新線束。
51.導線佈線:
      可使用自動和手動佈線模式，在保留對關鍵導線完全控制的同時，可快速佈置數千條導線。可以使用以下三種佈線功能將導線插入線束段中：
手動佈線，需要明確選擇佈線路徑。
交互式佈線，需要選擇佈線路徑的起點和終點，用計算法以供最短路徑的選擇。
自動佈線，基於所有可行路徑查找最短路徑。 
52.（新增功能）帶狀電纜:
      帶狀電纜能用三維數字模型表達，這就減少了電子設備的設計錯誤。在完全控制帶狀電纜扭曲位置和折疊位置的前提下，將帶狀電纜添加到設計中。 
53.XML輸出:
      XML輸出功能，用一個易於閱讀且與語言無關的XML文件完整描述了線束數據。並且可以將導線連接信息導入AutoCAD Electrical中，使得電路圖和導線圖的生成更加流暢。  
54.複製導線線束:
     線束設計的重用，可節省設計時間。將以前的設計作為原型，複製和重複使用現有線束部件。
55.生成報告:
     使用統一的數據存儲格式，優化並且自動創建報告。然後定義報告模式，運行報告，例如導線清單、終端圖表、剖面表，以及設計和製造線束的其他報告。 
56.虛擬零件:
    精確的物料清單意味著準確的採購和成本預算。只需要選擇線束部件（ 如夾子、端子、絕緣線束和標籤），然後將它們作為與Inventor 的物料清單完全集成的無圖零件插入電纜設計中即可。
57.計算線束直徑:
     使用三維可視檢查工具和乾涉檢測工具可以精確的確定導線線束是否適合於整個裝配。當添加或刪除導線時，軟件會自動計算線束直徑，甚至還會考慮到導線直徑以及線束之間必要的絕緣氣隙。
58.（新增功能）接頭:
      創建某公司特有的接插件庫，這有利於在設計中插入合適的接插件。Inventor提供了一個強大的接插件庫以簡化接插件的選擇和放置。同時資源中心提供了易於使用的編輯器和自定義接插件功能，以便於添加和修改接插件的屬性，例如接插件的零件編號和默認文件名。
59.多線電纜:
       跟踪各個電纜導線、導線插頭和佈線路徑，可以提高質量並且避免錯誤，並創建完整的物料清單，其中包括準確的電纜長度和數量，最大程度的減少浪費。為多線電纜創建智能演示，跟踪電纜中正在使用或者可以使用的導線，使用佈線以及取消佈線命令將電纜中所有導線捆在一起。
使用現有導線清單導入功能自動導入電纜。
創建可以精確顯示電纜數據的物料清單，而不是各條導線的物料清單。
創建電纜的可配置報告。
將DWG相關技巧合成到三維設計的工作流程中。以便更快捷地將零部件模型和工程圖數據結合在一起，並使得設計者與依賴於DWG技術的供應商和合夥人之間的交流更加順暢。這個機制是將Inventor中的工程圖存儲為DWG格式的文件，並且提供了真實的顯示表達、繪圖輸出和設計數據測量機制，同時完全保留了工程圖的關聯更新。 
60.（新增功能）打開DWG:
       文件在不安裝或打開AutoCAD的前提下直接打開二維設計的DWG 。在Inventor中直接打開AutoCAD的工程圖，並使用熟悉的Inventor環境實施查看、出圖和測量。用複制和粘貼命令，將現有的關聯的二維設計數據插入三維設計工作環境中。 
61.（新增功能）從Inventor 的視圖到AutoCAD 的塊:
       使用三維設計更新在原有二維環境中設計的項目，可以減少更新的成本。這個機制是將Inventor中的視圖轉換成AutoCAD的塊，因此可以在Inventor中重複設計部件，然後直接將新的工程圖視圖整合到原有的工程圖中。
62.零件設計:
      基於“功能設計”，Inventor幫助用戶定位設計的功能需求，以進一步創建三維模型，並節省設計具有競爭力產品的時間。 
63.（增強更能）草圖繪製:
     在創建零部件模型之前，需要對不同的設計決策進行評估。在Inventor的草圖環境中，能在二維環境中表達多種設計思路。通過簡明的約束工具，實現不同的設計方案的討論，草圖可有不同的顏色、線形和線寬來支持不同方案的表達。 
64.（增強功能）幾何結構分析:
      創建具有高質量曲面模型，並且檢測設計數據的可製造性，以降低製造成本。用於檢查設計幾何屬性的大量分析工具可以在零件設計環境和構造環境中使用。分析工具包括：
斑馬紋分析，增強了間距控制和顯示的準確性，完成對曲面相切和連續性的可視化確認。
高斯曲面分析，提供曲面曲率分析結果。
剖面分析，使用不同的顏色表達最大和最小壁厚的區域。
拔模斜度分析，使用不同的顏色表達基於開模方向的拔模角度，其中開模方向是由一條軸、一個面或者一個平面決定的。
檢測裝配中兩個成員或者兩個面之間的最小距離。 
65.（新增功能）鈑金件API:
     Inventor的應用程序接口 (API)提供了鈑金展開特徵以及沖壓庫，CNC自動化製造需要這些數據。因此可能把設計數據直接傳送到CNC上，加快從設計到製成品的過程。  
66.（新增功能）從Autodesk AliasStudio導入:
     可使用Autodesk AliasStudio的概念設計數據，以減少完成三維設計的時間。重用AliasStudio的曲線和曲面，這些信息來自這兩個產品內嵌的DWG導入和導出工具。在Inventor構造環境中實施曲面的導入，然後用灌注(Sculpt)特徵工具將這些曲面整合到三維零件模型中。  
67.灌注工具:
     使用灌注工具可以快捷地創建曲面圍成的形狀細節，這裡的曲面可以是在Inventor創建的，也可以是從其他軟件中導入的。還可以使用灌注工具添加或切除實體來編輯現有的模型，這就實現了基於一組封閉曲面或者從其他軟件中導入的曲面構建而成三維模型。

Creo (Pro/E)

開發起源

Pro/Engineer操作軟件是美國參數技術公司（PTC)旗下的CAD/CAM/CAE一體化的三維軟件。Pro/Engineer軟件以參數化著稱，是參數化技術的最早應用者，在目前的三維造型軟件領域中佔有著重要地位。Pro/Engineer作為當今世界機械CAD/CAE/CAM領域的新標準而得到業界的認可和推廣，是現今主流的CAD / CAM / CAE軟件之一，特別是在國內產品設計領域佔據重要位置。

主要特性
    Pro/E第一個提出了參數化設計的概念，並且採用了單一數據庫來解決特徵的相關性問題。另外，它採用模塊化方式，用戶可以根據自身的需要進行選擇，而不必安裝所有模塊。Pro/E的基於特徵方式，能夠將設計至生產全過程集成到一起，實現並行工程設計。它不但可以應用於工作站，而且也可以應用到單機上。
   Pro/E採用了模塊方式，可以分別進行草圖繪製、零件製作、裝配設計、鈑金設計、加工處理等，保證用戶可以按照自己的需要進行選擇使用。

1．參數化設計:
    相對於產品而言，我們可以把它看成幾何模型，而無論多麼複雜的幾何模型，都可以分解成有限數量的構成特徵，而每一種構成特徵，都可以用有限的參數完全約束，這就是參數化的基本概念。但是無法在零件模塊下隱藏實體特徵。
2．基於特徵建模:
    Pro/E是基於特徵的實體模型化系統，工程設計人員採用具有智能特性的基於特徵的功能去生成模型，如腔、殼、倒角及圓角，您可以隨意勾畫草圖，輕易改變模型。這一功能特性給工程設計者提供了在設計上從未有過的簡易和靈活。
3．單一數據庫（全相關）:
     Pro/Engineer是建立在統一基層上的數據庫上，不像一些傳統的CAD/CAM系統建立在多個數據庫上。所謂單一數據庫，就是工程中的資料全部來自一個庫，使得每一個獨立用戶在為一件產品造型而工作，不管他是哪一個部門的。換言之，在整個設計過程的任何一處發生改動，亦可以前後反應在整個設計過程的相關環節上。例如，一旦工程詳圖有改變，NC（數控）工具路徑也會自動更新；組裝工程圖如有任何變動，也完全同樣反應在整個三維模型上。這種獨特的數據結構與工程設計的完整的結合，使得一件產品的設計結合起來。這一優點，使得設計更優化，成品質量更高，產品能更好地推向市場，價格也更便宜  。

此內容管理系統以 https://github.com/mdecourse/cmsimde 作為 submodule 運作, 可以選定對應的版本運作, cmsimde 可以持續改版, 不會影響之前設為 submodule, 使用舊版 cmsimde 模組的內容管理相關運作.

利用 cmsimde 建立靜態網誌方法:

1. 在 github 建立倉儲, git clone 到近端

2. 參考 https://github.com/mdecourse/newcms, 加入除了 cmsimde 目錄外的所有內容

以 git submodule add https://github.com/mdecourse/cmsimde cmsimde

建立 cmsimde 目錄, 並從 github 取下子模組內容.

3.在近端維護時, 更換目錄到倉儲中的 cmsimde, 以 python wsgi.py 啟動近端網際伺服器.

動態內容編輯完成後, 以 generate_pages 轉為靜態內容, 以 git add commit 及 push 將內容推到遠端.

4. 之後若要以 git clone 取下包含 submodule 的所有內容, 執行:

git clone --recurse-submodules https://github.com/mdecourse/newcms.git

Develop

https://github.com/mdecourse/cmsimde 的開發, 可以在一個目錄中放入 cmsimde, 然後將 up_dir 中的內容放到與 cmsimde 目錄同位階的地方, 使用 command 進入 cmsimde 目錄, 執行 python wsgi.py, 就可以啟動, 以瀏覽器 https://localhost:9443 就可以連接, 以 admin 作為管理者密碼, 就可以登入維護內容.

cmsimde 的開發採用 Leo Editor, 開啟 cmsimde 目錄中的 cmsimde.leo 就可以進行程式修改, 結束後, 若要保留網際內容, 只要將 cmsimde 外部的內容倒回 up_dir 目錄中即可後續對 cmsimde 遠端倉儲進行改版.

init.py 位於  up_dir 目錄, 可以設定 site_title 與 uwsgi 等變數.
